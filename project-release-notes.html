Automated Software Update Tracking & Visualization

Project Overview

This project automates the tracking, analysis, and visualization of software updates across platforms like GitHub, Wikipedia, Reddit, and MitreCVE. It uses 10 intelligent bots that query data every four hours—resulting in over 60 queries per day and 1,800+ monthly. This enables real-time awareness of evolving software components and vulnerabilities.

The platform features a live dependency graph, an update feed sorted by relevance (chronology, security, legal), four analytical charts for insights, automatic UML architecture generation, and an AI chatbot for natural language querying. Together, they provide a complete view of software changes and system impacts.

With professionals often spending 20–30% of their time tracking changes manually, this system offers an estimated time saving of 10–15 hours per month per user. By prioritizing updates based on impact and dependency weight, it also reduces update risks and improves team alignment.

Importance of the Project

In IoT or cyber-physical systems, a missed or faulty update can lead to serious safety risks or large-scale system failure. Our solution helps reduce exploit windows by up to 70% by automating update detection and patch prioritization.

The tool benefits various roles: Requirements engineers can adapt specs to real-time changes; software architects avoid manual UML maintenance; developers are notified of relevant framework and library changes; testers know when to run regressions. This holistic coverage ensures reliable and secure software delivery.

Research & Insights

Our research found that most organizations rely on scattered, manual systems or limited tools tied to specific ecosystems. Academic work warns of dependency drift and patching gaps, which validated the need for a unified, proactive solution. We built our system around DevSecOps principles and security intelligence, integrating them deeply into the update process.

Takeaways

Visualizing component dependencies helped identify update-critical areas. Classifying updates by type and impact allowed better prioritization than by version number alone. Automating architecture diagram updates was one of the highest time-savers for teams.

Challenges

Aggregating data from platforms like GitHub and MitreCVE brought rate-limit, noise, and duplication issues. Building a clean graph model required custom logic. LLM-based querying needed context-aware prompts, and reliable UML generation demanded robust system parsing. Each of these added technical depth but brought high user value.
